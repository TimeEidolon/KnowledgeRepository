# 1. 功能简介

工艺编辑器支持企业十分自由地自定义工艺规则，同时最大程度降低功能开发和程序代码维护成本

工艺编辑器以台面线条领域为试点，辅助生成类模型（包括水槽，合并免拉手等），成功打通了多条功能链路，并持续拓展其应用场景和业务领域。下文会进行简单的介绍~

# 2. 工艺生成能力

自定义生成脚本是工艺编辑器的起点。初衷是为了解决不同商家对台面、线条、虚拟五金等生成模型有不同生成诉求，我们提供了一套「由商家自定义生成规则的能力」，允许通过工艺编辑器，编写外部脚本，自定义修改生成逻辑，从而做到个性化操纵生成模型的结果。有几种典型案例代表：

## 自定义生成虚拟五金

**常见问题：**

在台面线条把手等领域，除模型实体外，往往还需要增加五金配件。而什么产品加什么配件，加多少，什么型号，都是企业的计算规则，还与生产报价配货有关。以往通过设计师手动在面板上自行增加的形式，易错且增加设计负担，不利于管理

**功能优势：**

为了解决上述类似的问题，可以借助工艺编辑器，制定商家自定义的规则，配置主体与附属的关系，编写脚本，生成模型实体时，自动生成附属配件

## 自定义生成逻辑

**常见问题：**

在台面线条的实际生产中，不同的客户都有自己的生产规范。例如，线条多长打断，台面什么时候加厚，部分见光工艺等。这些定制化的工艺规则，会影响到最终生成模型的实际效果，比如有的商家要台面齐平，有的不要，有的线条要切割有的不要，所以我们无法用统一的生成规则来满足所有业务场景，特别是还存在互相冲突的规则时

**功能优势：**

为了解决上述类似的问题，可以借助工艺编辑器，在有限的范围内，调整生成逻辑，让模型生成效果的如脚本预设的那样

# 3. 工艺配置能力

**常见问题：**

大型企业对生产的把控严苛，什么工艺能做什么工艺不能做有固有的规定，所以会严格关注设计师画的模型的合规性，对于不能做的工艺是不能画出来的，能画出来就能生产。也因此，要求工具可以强约束设计师的设计行为

**功能优势：**

为了解决上诉类似的问题，可以借助工艺编辑器，预设一些条件去配置前端参数的可修改性，包括：显隐、置灰、可选项、值范围

# 4. 生成检测能力

即使我们对设计师的操作行为做了一些入口上的约束，但由于复改的动作，可能会导致当前的模型是不符合工艺要求的。因此需要对生成后的模型做检测，根据之前的「工艺配置」判断是否当前的模型是否正确。或者编写脚本，判断生成模型与其他模型之间的关系（例如水槽与台面扣件），提示设计师当前模型是否合规，并可做到下单拦截。与规则检测不同的是，工艺编辑器对编写复杂的几何计算脚本，及操纵生成模型上有着较为灵活的优势

# 5. 生成模型再加工

## 重定义生成模型的输出参数

**常见问题：**

生成模型是一次性买卖，即生成时确定生成脚本，生成完后模型的参数信息就已经确定了，并且各个生成模型之间是互相独立的。而在实际业务场景中，模型与模型之间、模型与环境之间是有关联的，这些关联可能与设计行为无关，仅仅是企业在输出和报价的时候，需要在模型上，计算并记录这些信息，便于与下游合作方对接。如果信息是生成的时候携带，就无法随着环境变化而变化，因此需要在更下游的时候，为模型添加这些关联信息

**功能优势：**

为了解决上诉类似的问题，可以借助工艺编辑器，编写关系脚本，在设计完成后，输出数据时，运行这个脚本，操作模型参数，目前支持：新增、修改

## 重定义生成模型的商品信息

**常见问题：**

生成类模型与普通的参数化模型，在创建商品时存在差异：参数化模型都是用户根据自己所需通过参数化编辑建立的具体商品，而生成类模型是Coohom预设的脚本，属于公共库隐藏商品，企业是无法自定义公库隐藏商品的商品信息。那么下游的后端软件，就无法通过识别产品编码等信息去对接模型

**功能优势：**

为了解决这个问题，可以借助工艺编辑器，编写映射表（生成参数对应的产品编码），并在生成模型时，预存模型对应的产品编码，从而在输出时可以修改模型对应的产品编码。目前支持修改的商品信息为：产品编码、型号

# 6. 新增自定义变量能力

**常见问题：**

生成模型脚本由于是Coohom预设的脚本，生成参数是不能让用户自定义的，而用户又需要新增一些自定义变量来表达生成模型的附属信息，这些变量还需要在前端设计师可见、可改、可输出

**功能优势：**

为了解决这个问题，可以借助工艺编辑器，对轮廓商品添加自定义参数
