Want your AI to directly use cabinet materials from the enterprise catalog and generate a perfect kitchen design plan with just one click? Simply complete a few steps to configure the products and templates, and your product library will instantly transform into an intelligent designer, allowing you to enjoy the magical experience of fast rendering and precise execution effortlessly!

### Step 4
Click **Add Type** in the upper right corner, and add products using the same method. * Add as many types as possible for better output results. * When selecting a type, each cabinet category can only be chosen once. * When choosing a category, select one category model cabinet from the enterprise library for each category (a category refers to one style, e.g., basic full-door cabinets. Do not select multiple styles of full-door cabinets. AI will choose cabinets according to the selected category, and mixed styles will greatly affect the output results).

### Step 5
After adding the products, the products will be placed in the list below. At this point, there are different configuration methods for various library-building approaches. * **Example 1:** I am a standardized cabinet enterprise; my cabinets are built individually, and there are many for one style, differing only in size (commonly used in North America). Configuration is as follows: + Select all cabinets of desired sizes under the desired category here. + Since each cabinet currently has a fixed size, there's no need to enable **ScalE**. * **Example 2:** I am a custom cabinet enterprise; I can produce any cabinet, and AI can randomly generate cabinet widths within my processing range (suitable for most companies). Configuration is as follows: + First, select corresponding models within the enterprise catalog. + Enable **Scale**. If you hover over the info icon behind it, you'll see the width range for the cabinet set in the parametric editor. + Simply enable **Scale**, meaning the current cabinet's width range corresponds to the parametric editor's range, allowing AI to generate values within it. * **Example 3:** I am a custom cabinet enterprise; although my cabinets are custom-made, I have certain cabinet width preferences (suitable for most companies). Configuration is as follows: + After enabling **Scale**, users can input the desired width value in the text box behind it (note: this width value supports manual input or dropdown selection, depending on user model configuration; if the model is configured as a numerical range, input any value within that range; if the model is configured as fixed options, select the corresponding option via dropdown). + Width values must be within range and cannot exceed parameter limits.

![](https://qhstaticssl.kujiale.com/image/png/1778665501350/52B50AEBD486558B11C88588CDAC9547.png)

2. In the custom product template page, **Add Template** according to the configuration (up to 10 templates can be created). You can also **Rename** the templates.

![](https://qhstaticssl.kujiale.com/image/png/1778665529573/33F76B624F087D5FBE66B2500A1234DC.png)

![](https://qhstaticssl.kujiale.com/image/png/1778666205165/892BF8050D8C9C8B2D91E4EE7154E170.png)

![](https://qhstaticssl.kujiale.com/image/png/1778666229598/0E0A8D330D758AC7D82F7100A7B88D9B.png)

* When setting categories, all configurations made can be selected. Different templates may reuse the same configurations.
