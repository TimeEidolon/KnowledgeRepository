### Step 7
After adding the products, the products will be placed in the list below. At this point, there are different configuration methods for various library-building approaches. * **Example 1:** I am a standardized cabinet enterprise; my cabinets are built individually, and there are many for one style, differing only in size (commonly used in North America). Configuration is as follows: + Select all cabinets of desired sizes under the desired category here. + Since each cabinet currently has a fixed size, there's no need to enable **ScalE**. * **Example 2:** I am a custom cabinet enterprise; I can produce any cabinet, and AI can randomly generate cabinet widths within my processing range (suitable for most companies). Configuration is as follows: + First, select corresponding models within the enterprise catalog. + Enable **Scale**. If you hover over the info icon behind it, you'll see the width range for the cabinet set in the parametric editor. + Simply enable **Scale**, meaning the current cabinet's width range corresponds to the parametric editor's range, allowing AI to generate values within it. * **Example 3:** I am a custom cabinet enterprise; although my cabinets are custom-made, I have certain cabinet width preferences (suitable for most companies). Configuration is as follows: + After enabling **Scale**, users can input the desired width value in the text box behind it (note: this width value supports manual input or dropdown selection, depending on user model configuration; if the model is configured as a numerical range, input any value within that range; if the model is configured as fixed options, select the corresponding option via dropdown). + Width values must be within range and cannot exceed parameter limits.

MEDIA_STEP_7_TEXT: After adding the products, the products will be placed in the list below. At this point, there are different configuration methods for various library-building approaches. * **Example 1:** I am a standardized cabinet enterprise; my cabinets are built individually, and there are many for one style, differing only in size (commonly used in North America). Configuration is as follows: + Select all cabinets of desired sizes under the desired category here. + Since each cabinet currently has a fixed size, there's no need to enable **ScalE**. * **Example 2:** I am a custom cabinet enterprise; I can produce any cabinet, and AI can randomly generate cabinet widths within my processing range (suitable for most companies). Configuration is as follows: + First, select corresponding models within the enterprise catalog. + Enable **Scale**. If you hover over the info icon behind it, you'll see the width range for the cabinet set in the parametric editor. + Simply enable **Scale**, meaning the current cabinet's width range corresponds to the parametric editor's range, allowing AI to generate values within it. * **Example 3:** I am a custom cabinet enterprise; although my cabinets are custom-made, I have certain cabinet width preferences (suitable for most companies). Configuration is as follows: + After enabling **Scale**, users can input the desired width value in the text box behind it (note: this width value supports manual input or dropdown selection, depending on user model configuration; if the model is configured as a numerical range, input any value within that range; if the model is configured as fixed options, select the corresponding option via dropdown). + Width values must be within range and cannot exceed parameter limits.
