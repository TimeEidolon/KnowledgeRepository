In this article, we will analyze the frequenly asked issue of failing to add manual template while rendering the project and introduce the way to resolve it.  

## Users: All Users


## **Issue**:

As long as users add more than 20 manual lighting templates, they will get the error message.

" *Failed to add manual lighting template. The maximum number of manual lightings has been reached, which is 20. Please delete some manual lighting templates before adding more*"

Please be noted that this is not a bug but a warning message to remind users of the tool limit and how to get it resolved. 

![](https://qhstaticssl.kujiale.com/image/png/1761638808764/A8CEBABD0663ECBAEEE7516DF7A7D2B5.png)

However, users may get confused regarding the resolution and will request Coohom to enlarge the limit.

## Reason:

This is because users will tend to add new lighting template each time to enter once they exit the lighting customization interface. This accumulated adding of the same template will lead to the maximum number to trigger the alert message.

![](https://qhstaticssl.kujiale.com/image/png/1761640817915/A8CEBABD0663ECBAEEE7516DF7A7D2B5.png)

![](https://qhstaticssl.kujiale.com/image/png/1761641720385/A8CEBABD0663ECBAEEE7516DF7A7D2B5.png)

## Resolution

Step 1: Delete the duplicate lighting templates by hovering the mouse to the lighting template to show the delete button. (PS: only one lighting template is working at a time in render.)

![](https://qhstaticssl.kujiale.com/image/png/1761642200672/A8CEBABD0663ECBAEEE7516DF7A7D2B5.png)

Step 2：Entering the same lighting template each time to continue the lighting customization once exited.

![](https://qhstaticssl.kujiale.com/image/png/1761642290696/A8CEBABD0663ECBAEEE7516DF7A7D2B5.png)
