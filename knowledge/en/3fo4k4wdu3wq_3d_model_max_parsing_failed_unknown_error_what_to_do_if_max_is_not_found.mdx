# Problem 1

Cannot find max

# Solution 1

Extract locally and open the folder to check if there is a max file. In most cases, this is caused by uploading files of other types to the model entry.

# Problem 2

MAX parsing failed

# Solution 2

1. Max compressed package exceeds 200MB, such as uploading an entire space (e.g., a bedroom), this requires splitting the model.

2. Exceeds model face limit (exceeds platform 2 million face limit).

3. Existence of empty objects and auxiliary objects, and too many of them, leading to parsing failure. If you cannot find the auxiliary objects yourself, export to FBX format file and then open to see them.

4. MAX file cache is too much or the MAX file has a virus.

5. Check if the unit setting of the MAX file is mm, if not, set the unit to mm, and re-upload to see if it succeeds.

# Problem 3

Unknown error

1. Compressed file has extraction password.

2. Too many comprehensive model issues.

3. Network issues, re-upload can solve it.

# Solution 3

If the above situations cannot be solved by the business, you can consult an online model expert or find the corresponding model operation to solve the problem.
