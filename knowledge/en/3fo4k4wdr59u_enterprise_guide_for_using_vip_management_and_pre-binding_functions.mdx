Enter **Workspace**, select **"Account Management - VIP Management"**, click the upper right corner of the privilege management page **"Privilege Pre-binding"**. * **Note:** This function is only visible to the primary administrator account. Click **Create a Pre-bound Rule**, complete the filling according to the suggestions, and click confirm after completing, required information includes: * **Rule Name**: Used only to distinguish between different rules. * ​**Rule Validity Period**: Define a specific duration for the rule to be effective, or create a permanent rule. * ​**Scope of Effect**: Apply the rule to all accounts or only to a specified subset of accounts. * **Pre-bound** ​**VIP Type**: Select from the VIP types available in the current Coohom version. After a rule is created, whenever an account's VIP status expires or a new VIP is purchased, the system will automatically check if any accounts meeting the rule's conditions have lost their VIP status and automatically rebind the VIP accordingly, saving time and effort!

Coohom enterprise account administrators

# Prerequisites

The enterprise primary administrator has VIP management functionality enabled by default, while other personnel do not have this function for the time being.

# VIP Management Function Introduction

## Entrance

Enter **Workspace**, select **“Account Management - VIP Management”**, and enter the **Privilege Management** page.

![](https://qhstaticssl.kujiale.com/image/png/1762307823292/A8CEBABD0663ECBAEEE7516DF7A7D2B5.png)

## View by VIP Status

After entering VIP management, you can view VIPs in different statuses, current bindings, and usage rates. The statuses are as follows:

* **Privileges Overview**: Displays the total number of privileges and the number in use (e.g., 1/25, where 1 is in use and 25 is the total quantity).
* **Bound**: VIPs that are already bound.
* **Unbound**: VIPs that are not yet bound.
* **Renewal Plan**: VIPs set to be automatically inherited by another designated account after the current VIP expires.
* **Expires in 30 Days**: Privileges that will expire within 30 days.

After selecting a specific status, the list below will load the corresponding VIP details. This detailed information typically includes the VIP source, status, bound object (nickname, email), and expiration time.

![](https://qhstaticssl.kujiale.com/image/png/1762308160686/A8CEBABD0663ECBAEEE7516DF7A7D2B5.png)

# VIP Pre-binding Function Introduction

The VIP pre-binding function can automatically bind a type of VIP after some accounts' VIPs expired, achieving the effect of perpetual VIP status.

## Entrance

Enter **Workspace**, select **"Account Management - VIP Management"**, click the upper right corner of the privilege management page **"Privilege Pre-binding"**.

* **Note:** This function is only visible to the primary administrator account.

![](https://qhstaticssl.kujiale.com/image/png/1762310151214/A8CEBABD0663ECBAEEE7516DF7A7D2B5.png)

## Rules Configuration and Viewing

Pre-binding is a function based on the configuration and execution of rules, requiring the relationship between **accounts and effective rules** to be set in advance.

Click **Create a Pre-bound Rule**, complete the filling according to the suggestions, and click confirm after completing, required information includes:

* **Rule Name**: Used only to distinguish between different rules.
* ​**Rule Validity Period**: Define a specific duration for the rule to be effective, or create a permanent rule.
* ​**Scope of Effect**: Apply the rule to all accounts or only to a specified subset of accounts.
* **Pre-bound** ​**VIP Type**: Select from the VIP types available in the current Coohom version.

![](https://qhstaticssl.kujiale.com/image/png/1762310269155/A8CEBABD0663ECBAEEE7516DF7A7D2B5.png)

![](https://qhstaticssl.kujiale.com/image/png/1762310357290/A8CEBABD0663ECBAEEE7516DF7A7D2B5.png)

After a rule is created, whenever an account's VIP status expires or a new VIP is purchased, the system will automatically check if any accounts meeting the rule's conditions have lost their VIP status and automatically rebind the VIP accordingly, saving time and effort!
