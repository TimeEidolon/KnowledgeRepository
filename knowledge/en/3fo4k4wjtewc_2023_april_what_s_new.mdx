### **【Custom Order】Custom order support configuring Not Allowing Modification when resubmitting**

### **Introduction**

Custom orders provide the ability for merchants to configure custom fields for order review, and some custom fields may be used by merchants for system docking and feedback writing. If the value of a field that is written back is modified when resubmitted, it may cause data errors. Therefore, this update adds a configuration item that allows custom fields to be configured to not allow manual modification when resubmitted.

### **Aim User**

For part of Enterprise customers.

### **Specific Operation**

1. Enter the custom order management background → order settings → custom information, click "Edit" or "Edit Field", and you can find a new configuration item 【Allow Modification】.

Enter the custom order management background → order settings → custom information, click "Edit" or "Edit Field", and you can find a new configuration item 【Allow Modification】.

![](https://qhstatic-us-s3.coohom.com/yuntai/image/png/1733842174192/A36EC22D7FB823E938183E9869096C5B.png)

2. Configuring it as 【No】 means not allowing modification. In the tool, when resubmitting, the field will be grayed out and cannot be manually modified.
