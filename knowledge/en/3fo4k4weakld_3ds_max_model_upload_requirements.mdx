To facilitate a smoother upload of 3Ds max models in Coohom, this article will provide you with detailed instructions on the upload requirements for 3Ds max models.

# For Users.

All users

# Supported Formats

1. 2021 and earlier versions
2. .max format, can be uploaded directly, if there are textures, they need to be archived and packed into zip or rar format, archiving method: (link)
3. .3ds / .dae / .prt / .asm formats need to be packed into zip or rar

# Model Requirements

| Category | Specific Requirements | Precautions |
| --- | --- | --- |
| Model Size | Unit set to millimeters (mm), size within 30000mm\*30000mm | 1) Only one **.max** source file can be included in a compressed package.  2) The compressed package cannot have a decompression password set.  3) The **.max** source file must not be damaged and must be able to be normally opened with 3ds Max.  4) The source file must not contain third-party scripts such as **ALC, ALC2, CRP, ADSL**.  5) The source file must contain a model, uploading an empty scene is not allowed.  6) A model file that is too large and with too many triangular faces will cause extended upload time, rendering failure, and surface breaking.  7) Different materials must be separately attached to different parts of the model, multi-dimensional sub-materials must be split for easy material replacement later.  8) Emissive materials and VRay lights are supported, but when using VRay lights, ensure they are needed for the product (fixtures, lightboxes, etc.), but auxiliary lighting used for rendering must be removed.  9) Avoid changing the texture size in the material editor, do not use real-world scale for textures and cropping |
| Model Size | Within 200MB |
| Model Faces | Within 2 million triangular faces |
| Model Objects | Within 200 OBJ parts |
| Objects in Scene | Only retain the necessary model itself, delete cameras and other auxiliary objects, if retaining light sources is needed, unfreeze and unhide them before uploading together |
| Model Orientation | In the front view of max, the front of the model should face forward to avoid the wrong default angle after uploading. |
| Model Modifiers | If the model has multiple stacked modifier commands, it is recommended to right-click and convert the model to an editable polygon/edit mesh. |
| Model Material | VRay material (VRayMtl) |
| Model Textures | Texture file size within 5MB  Pixel size within 5000x5000 and not less than 1000x1000  Supported formats: jpg / png / tga, color mode is RGB  Textures can only use channel 1 and do not select real-world scale |
| Gamma Value Requirements | Select Enable Gamma/LUT Correction, set max file Gamma to 2.2;  Open File - View Image File, ensure all textures are set to Auto (recommended) or Override 2.2 |

# Material Support Status

Most materials are supported for upload, but the following textures and materials are not supported: ColorCorrect, VaryFastSSS2, Vraysky, Perlin Marble, etc.

For detailed material upload status: https://www.coohom.com/helpcenter/max-model-material-upload-status

For detailed mapping upload status: https://www.coohom.com/helpcenter/max-model-mapping-upload-status
