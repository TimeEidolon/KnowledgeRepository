Want to quickly master the use of the real-time material tool? This article will provide you with a detailed explanation of the main steps for using the real-time material tool: select a material template —> replace/upload material maps (optional) —> adjust material parameters —> render material effects —> save/batch save materials. Come read the article and learn step by step how to use the real-time material tool to create the ideal effect!

### Step 2
Enter the new material page, click **Real-time Material.** The main steps of using the real-time material tool are: select a material template —> replace/upload material maps (optional) —> adjust material parameters —> render material effects —> save/batch save materials. In the left toolbar, you can select different types of material templates, including board, stone, plastic, glass, leather, and more. You can choose according to your needs. After selecting a material template, you can also choose to replace the system map or upload your own material map. **1. Replace System Map** **2. Upload Material Map** In the right panel, you can modify material reflection and diffuse parameters, and you can view the effects in real-time when adjusting. * **Diffuse:** color and texture details of the material surface. * **Reflection:** color of the material reflecting light. * **Reflection Glossiness:** controls the clarity of reflection on the surface of the object; the larger the value, the clearer the reflection. * **Bump:** bump condition of the material surface, white areas are raised, black areas are indented. After completing the material parameter settings, you can click **Render**, choose different scenes to render, and preview the material effects in different scenes. **1. Save Materials** After creating the material, you can save the material directly. After clicking the **Save** button, you need to name the material, set the material size, choose the rendering category, and select the save folder. **2. Batch Save Materials** Click **Batch Save**, and you can save the material as different rendering categories based on the material parameters you set this time. Enter the batch save materials page, select the render category, choose according to the format required for upload **Click Upload**, and after selecting, click **Save.** Enterprise users can save the material as a template, click **Save as Template**, set the template name and folder. **Q1: Can there be any difference in using various materials made with the Real-time Material Editor?** **A1:** No, there’s no difference in using the materials made with the Real-time Material Editor. In our design tools, you can replace the materials, render them, and perform more operations in the same way, regardless of the material. In the Sass backend, you can manage and edit them all as expected. **Q2: Do the materials made with the Real-time Material Editor have the same rendering effect as Coohom projects or floor plans?** **A2:**The rendering effects are basically the same. However, there can be slight visual differences due to varied settings of scenes, lighting, and camera angles. **Q3: What is Material Template?** **A3:**Material Template is the updated version of Rendering Category. Each template can be seen as a material base with pre-configured basic effects. By adjusting parameters, you can edit on the material base to create a new material on your own. **Q4: If I forget to save a material that has been rendered or adjusted, will that material still be there after I exit the page?** **A4:**No, your rendered or adjusted material cannot be retrieved if you quit without saving. We recommend that you save it soon after the rendering or adjustment is done. **Q5: Will the effects of a real-time material look exactly the same as before if I finally set its parameters and category to the original state?** **A5:**In most cases, it will look exactly the same as before. However, there can be trivial differences in some materials. So we suggest that you observe the actual effects after setting the parameters and category to the original state to make sure it meets your expectation. **Q6: Why would renderings fail sometimes?** **A6:**Renderings could fail sometimes because of low internet speed or network congestion. You can click Retry after a while for a new attempt. **Q7: Is there a limited number of renderings that I’m entitled to in the material editor?** **A7:**So far, there’s no limit to the renderings entitled to any user in the material editor. However, if you start too many rendering operations in a short period, the tasks may end up in a long queue, prolonging the rendering process greatly. Therefore, we suggest that you cut back on the renderings that are not so necessary.

# FAQ

**Q1: Can there be any difference in using various materials made with the Real-time Material Editor?**

**A1:** No, there’s no difference in using the materials made with the Real-time Material Editor. In our design tools, you can replace the materials, render them, and perform more operations in the same way, regardless of the material. In the Sass backend, you can manage and edit them all as expected.

**Q2: Do the materials made with the Real-time Material Editor have the same rendering effect as Coohom projects or floor plans?**

**A2:**The rendering effects are basically the same. However, there can be slight visual differences due to varied settings of scenes, lighting, and camera angles.

**Q3: What is Material Template?**

**A3:**Material Template is the updated version of Rendering Category. Each template can be seen as a material base with pre-configured basic effects. By adjusting parameters, you can edit on the material base to create a new material on your own.

**Q4: If I forget to save a material that has been rendered or adjusted, will that material still be there after I exit the page?**

**A4:**No, your rendered or adjusted material cannot be retrieved if you quit without saving. We recommend that you save it soon after the rendering or adjustment is done.

**Q5: Will the effects of a real-time material look exactly the same as before if I finally set its parameters and category to the original state?**

**A5:**In most cases, it will look exactly the same as before. However, there can be trivial differences in some materials. So we suggest that you observe the actual effects after setting the parameters and category to the original state to make sure it meets your expectation.

**Q6: Why would renderings fail sometimes?**

**A6:**Renderings could fail sometimes because of low internet speed or network congestion. You can click Retry after a while for a new attempt.

**Q7: Is there a limited number of renderings that I’m entitled to in the material editor?**

**A7:**So far, there’s no limit to the renderings entitled to any user in the material editor. However, if you start too many rendering operations in a short period, the tasks may end up in a long queue, prolonging the rendering process greatly. Therefore, we suggest that you cut back on the renderings that are not so necessary.
