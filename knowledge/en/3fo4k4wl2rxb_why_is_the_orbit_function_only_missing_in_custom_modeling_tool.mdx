Recently one Coohom user has reported an issue that the mouse's orbit function is working fine in other Coohom tools except in custom modeling tool.

![](https://qhstatic-us-s3.coohom.com/yuntai/image/png/1742392574447/464F026B420D05F50382A202B64EF83D.png)

![](https://qhstatic-us-s3.coohom.com/yuntai/image/png/1742392579477/159F8648317353C3630F04D371206334.png)

This is because he accidentally changed the mode in the custom modeling tool from **Default template** to **Sketchup teamplate**. Here we will introduce the two functions respectively for the two templates. 

**Default template**- The orbit function of the mouse is showing normally in order to rotate the floor plan in the canvas like that in the other tools.

**Sketchup template**-The orbit function will only show while pressing mouse wheel to operate.

Then, where and how to change this function as per users' preference?

Here is the route: Help icon>Shortcut keys and mouse>Current version

![](https://qhstatic-us-s3.coohom.com/yuntai/image/png/1742392583017/EF7866CAB912D2876A0608D2B5DD65D1.png)

Hence, users can select the template which he or she prefers in order to make orbit function work smoothly. For any queries or suggestions, please feel free to contact **Customer Service** team via **support@coohom.com**.
