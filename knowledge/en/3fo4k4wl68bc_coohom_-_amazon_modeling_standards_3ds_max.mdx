### **1.1.** **Models must comply with the Coohom upload standards.**

### **1.2.** **Model Size**

The file size of the model should be within 200M, not exceeding 200 geometries, and the number of faces should be within 2 million.

*To ensure smooth 3D preview on Amazon, the number of model faces should not exceed 200,000 triangles.*

### **1.3.** **Basic Settings**

The unit is set to millimeters (mm);

The models within the scene should be controlled within 30 meters by 30 meters, keeping the z-axis as the upward axis.

The settings screenshot is shown in Figure 1; [Image]

### **1.4.** **UV**

The model UV should be within the range of 0-1, and the unfolding size should be determined by the size and detail level of the model part, and should be distributed reasonably.

The UV rotation direction must match the actual application, and there should be no texture confusion. The arrangement example is shown below;

![](https://qhstatic-us-s3.coohom.com/yuntai/image/png/1745245062211/3E121190D35515E9B42E7BB26A2E4875)

#### **2.** **Modeling Specification**

#### **2.1.** **Ensure consistency between reference materials and 3D models, as Amazon will manually review color, shape, outline, and modeling details.**

*Reference Image: The main product image serves as the primary reference for comparison. Unless there is a conflict between the product description and other photos and the main image, it is considered that the main image is incorrect. In all other cases, other material information (3D model) should be reviewed according to the main product image.*

#### **2.2.** **Switch State**

Regardless of the open and closed state of the product in the main image, it is recommended to keep the model closed, with power cords and other items placed at the back of the product (example shown in below);

### **2.4.** **Color**

The model must be completely consistent with the main product image in the product ASIN, and models of the same style in other colors are not accepted. The model color must be consistent with the reference image, such as the reflective effect of the leather material and the natural damage of the leather, which should be accurately reflected. If the auxiliary images or product descriptions conflict with the main image, causing the reviewers to believe that the "main image" may be inaccurate, the 3D model release may be delayed or rejected.

### **2.5.** **Shape**

The outline of the 3D model must match the outline of the reference physical image. Any visible differences are considered to be of poor model quality; for example, if a sofa has natural wrinkles and unevenness, it should not be beautified or modified during the modeling process and should be accurately reflected. The 3D model has differences from the reference image, and the release of the 3D model may be delayed or rejected.

### **2.6.** **Material Expression**

The smoothness of the material, the bump effect, etc., must be the same as the reference physical image. The smoothness and depth of the folds of the 3D model, and the smoothness of the inclined surface, are different from the reference image.

### **2.7.** **Details**

All external details of the product that can be seen must be displayed in the 3D model, including buttons, fabric folds, product gaps, fine metal accessories, various material details.

The sewing method of the fabric, the arrangement of the nails, the connection method of the components, the splicing method of the combination, including all functional hardware elements (such as screws, nuts, bolts, buttons, hinges, pins, brackets, supports) that can be seen on the main detail page, should be correctly modeled, as shown in below.

### **2.8.** **Content Exclusivity**

If there is an inconsistency in the content between the main image on the product detail page and other images, model according to the main image (example shown in below).

![](https://qhstatic-us-s3.coohom.com/yuntai/image/png/1745245066481/C2D61385CBC7469678E901A4F2013769)

### **3.** **Material Requirements**

### **3.1.** **Coohom only supports the VRay renderer.**

### **3.2.** **Supported Material Range:**

**VRay Standard Material (VRayMtl)**

Supported Material Nodes: Opacity, Refraction, Diffuse, Bump, Reflection, Reflect Glossiness, Fresnel, Fresnel IOR Lock, Fresnel IOR, Refraction IOR All the above material nodes only support texture/color form (as shown in below), if connected to other program textures, there will be a loss of effect.

![](https://qhstatic-us-s3.coohom.com/yuntai/image/png/1745245066858/E9A1B3DE36CCD91B1B71DBD0CE530378)

**VRay Blend Material (VRayBlendMtl)**

![](https://qhstatic-us-s3.coohom.com/yuntai/image/png/1745245067130/1579FC6ECBC805532657268526E45872)

### **3.3.** **Texture Requirements**

The texture file pixel should be within ***4096\*4096****, and not lower than* ***2048\*2048***. Textures should use .jpg .png, and the color mode should be RGB.

#### **4.** **Model Upload**

The model location should be selected according to the actual situation of the product model (ground, ceiling, wall model). [Image]
