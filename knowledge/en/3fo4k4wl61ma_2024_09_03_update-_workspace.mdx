### Step 1
**Target Users** All enterprise clients **Before** Previously, when enterprise users uploaded models and textures to Coohom, it was not possible for them to save these materials as multiple copies for backup. Users could only re-upload the same model and texture from Coohom. **After** Now, enterprise users can directly select the materials they wish to back up in bulk and then save them as new. **How to use** Users can go to the enterprise library, select items in bulk, copy them as new items, and place them into the corresponding folders.

### Step 3
**【Workspace】Supports for enterprises to modify rendering categories for real-time materials** **Target Users** All enterprise clients **Before** Previously, when enterprises used the real-time material editor to create and save materials, they were unable to modify the rendering categories of the materials once saved. If the rendering category was set incorrectly, they had to redo the material creation process. **After** Now, enterprise users can directly modify the rendering categories of real-time materials without having to create a new material again. **How to use** Users can go to the enterprise library, access the details of a material (real-time material), click on "Material category," and make the necessary modifications.

**【Workspace】Supports for enterprises to modify rendering categories for real-time materials**

**Target Users**

All enterprise clients

**Before**

Previously, when enterprises used the real-time material editor to create and save materials, they were unable to modify the rendering categories of the materials once saved. If the rendering category was set incorrectly, they had to redo the material creation process.

**After**

Now, enterprise users can directly modify the rendering categories of real-time materials without having to create a new material again.

**How to use**

Users can go to the enterprise library, access the details of a material (real-time material), click on "Material category," and make the necessary modifications.
