# Function Overview

1. The model splitting tool now supports nested combination capabilities for parts, allowing users to freely edit the part combination relationships within the tool.

2. It also supports importing hierarchical structure information and part names of different nodes created in local modeling software.

# Feature Introduction

**Note:**

**a. Currently, hierarchical structure editing is only supported within the model splitting tool.**

**b. Models with animations set cannot be split; animation data must be cleared first.**

## Basic Features

**1. Creating part combination relationships:** Currently, there are three ways to create part combinations.

1. Drag-and-drop creation: Only supports dragging a single part or complete part combination into a combination node, not directly into a single part.
2. Multiple selection shortcut creation: Select multiple parts/combinations and use the shortcut key commd/alt+G to create combinations.
3. Combination button creation: Select multiple parts/combinations and click the 【combination】button to create combinations.

![](https://qhstaticssl.kujiale.com/image/gif/1769757693725/3161E3004791C00731893E3D5736087C.gif)

**2. Resetting part combination relationships:** You can modify and reset part combination relationships through drag-and-drop operations in the hierarchy panel.

**Note: Use the white highlight guide line during dragging to determine the current hierarchy level.**

For example, in the video below, parts are placed inside or outside combinations based on the white highlight guide line.

![](https://qhstaticssl.kujiale.com/image/gif/1769757998676/9BFF424A3AC4EB463BB9DD4F04872EC6.gif)

**3. Disconnecting part combination relationships:** Currently, there are three ways to disconnect part combinations.

1. Drag-and-drop ungrouping: Suitable for any node, including single parts and part combinations.
2. Shortcut key ungrouping: Select multiple parts/combinations and use the shortcut key commd/alt+Shift+G to ungroup.
3. Unmarshal button: Select multiple parts/combinations and click the 【Unmarshal】button to ungroup.

**4. Model splitting:** After splitting, the hierarchical structure's **first level** data is automatically converted into corresponding sub-models.

For example, in the images below, the data at the first level hierarchy contains three items. After splitting, the resulting combination model contains three sub-models.

![](https://qhstaticssl.kujiale.com/image/png/1769758212390/A8CEBABD0663ECBAEEE7516DF7A7D2B5.png)
